## 6. Data Quality Summary
- Dataset is balanced across sentiment classes.
- HTML tags (mainly <br />) are pervasive and should be stripped.
- Very few reviews are empty; content is long-form and information-rich.
- Duplicate texts exist but conflicting labels are rare.
- Minimal URL noise and manageable non-ASCII usage.

## 7. Preprocessing Recommendations
- Remove HTML tags/markup and normalize whitespace.
- Lowercase text but retain punctuation conveying sentiment (e.g., !?).
- Do not remove negation tokens (not, never, no); they drive polarity cues.
- Optionally filter common stopwords but keep sentiment carriers and negations.
- Drop exact duplicate reviews before train/validation splitting if needed.

## 8. Feature Engineering Recommendations
- Use TF-IDF unigrams as a baseline and extend to bigrams to capture phrases (e.g., 'not good').
- Add review length or word-count features to capture verbosity/emphasis patterns.
- Include punctuation-based cues (exclamation count, question marks) for classical models.
- Consider simple negation propagation features (tag tokens following a negation).
- Dimensionality reduction (e.g., Truncated SVD) is optional; start without it and introduce if models overfit.
